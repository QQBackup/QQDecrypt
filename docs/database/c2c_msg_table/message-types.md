# 消息类型与属性标志

`c2c_msg_table` 中判断消息类别，主要依赖三个外层字段：

- `40011`：消息内容类型。
- `40012`：消息属性或子类型；只有部分场景是位掩码。
- `40013`：消息发送方向。

详细统计以目标仓库的 [`40011.md`](https://github.com/QQBackup/ntqq_msg_db_util/blob/master/db_docs/c2c_msg_table/40011.md)、[`40012.md`](https://github.com/QQBackup/ntqq_msg_db_util/blob/master/db_docs/c2c_msg_table/40012.md) 和 [`40013.md`](https://github.com/QQBackup/ntqq_msg_db_util/blob/master/db_docs/c2c_msg_table/40013.md) 为准。

## `40011`：消息内容类型

| 值 | 类型 | 主要特征 |
| --- | --- | --- |
| `0` | 无效或已删除消息 | `40800` 通常为 NULL |
| `1` | 特殊协议消息 | 样本较少，语义尚未完全确认 |
| `2` | 标准消息 | 文本、图片、表情等，`40012` 作为位掩码进一步区分 |
| `3` | 文件传输 | `40012` 使用文件类型 bit 标志 |
| `5` | 名片或引用回复 | 可能包含名片和被引用消息 |
| `6` | 名片/联系人分享 | 通常为名片信息及头像数据 |
| `7` | 带位置 tag 的短视频 | 最新研究确认不是地图位置消息 |
| `8` | 旧协议合并转发 | `40800` 内为 XML 描述 |
| `9` | Reaction | `40012` 通常为 `0x21`，并配合 `40850/40851` |
| `10` | 视频或音视频流记录 | 样本很少，`content_type=9` |
| `11` | 合并转发 | NT structmsg 或批量转发 |
| `17` | 消息状态通知 | `40012` 通常为 `0x0008` |
| `19` | 通话通知 | `40012` 镜像 `call_type` |

### `40011=2` 时的 `40012` 位掩码

| 掩码 | 含义 | 说明 |
| --- | --- | --- |
| `0x0001` | 含文本 | 通常对应 `40800` 内 `content_type=1` |
| `0x0002` | 含富媒体 | 通常对应图片或表情等 `content_type=2` |
| `0x0010` | 撤回/状态变更 | 消息体通常较小，具体语义仍有部分推测 |
| `0x1000` | 长消息/合并转发 | 消息体明显变大；仅在 `40011=2` 下出现 |

常见组合：

| `40012` | 常见解释 |
| --- | --- |
| `1` | 纯文本 |
| `2` | 图片或其他富媒体 |
| `3` | 图文混排 |
| `16` | 撤回或状态变更 |
| `33` | 引用回复 |
| `4096` | 长消息或合并转发 |

> `40011≠2` 时，`40012` 通常不是自由组合的 bitmask，而是该消息类型绑定的子状态值。例如文件传输使用文件类型码，Reaction 常见值为 `0x21`，通话通知则直接镜像 `call_type`。

## `40013`：消息方向

| 值 | 含义 | 联动判断 |
| --- | --- | --- |
| `0` | 对方发送，收到的消息 | 通常 `40033 == 40030` |
| `1` | 当前账号正常发出 | `40033` 通常为当前账号 QQ |
| `2` | 当前账号发出的特殊类型 | 发送者仍是当前账号 |
| `3` | 系统消息/通知 | `40030` 与 `40033` 通常为 `0` |
| `4` | 极少量自发特殊状态 | 可能与撤回或发送失败有关 |
| `5` | 极少量自发特殊状态 | 常与批量转发相关 |

旧版迁移数据中还可能出现整日 Unix 时间戳形式的大整数，这类值属于迁移噪音，不能当作正常方向枚举。

## 通话通知（`40011=19`）

此类消息中 `40012` 与通话类型一致：

| `call_type` / `40012` | 含义 |
| --- | --- |
| `3` | 对方来电但未接听 |
| `7` | 通话接通，`48152` 保存时长（毫秒） |
| `8` | 对方来电未接听的另一种类型 |
| `13` | 已在其他设备处理 |
| `33` | 远程协助，带通话时长 |
| `34` | 当前账号主动取消 |

## 文件传输类型码（`40011=3`）

`40012` 的单 bit 值可用于区分文件类型：

| 掩码 | 文件类型 |
| --- | --- |
| `0x1` | 文档或 PDF |
| `0x2` | 图片 |
| `0x4` | 视频 |
| `0x8` | 音频 |
| `0x10` | Word |
| `0x20` | PPT |
| `0x40` | Excel |
| `0x200` | 压缩包 |
| `0x800` | 可执行文件 |
| `0x8000` | GIF 动图 |

## 位置消息的修正

目标研究对 `40011=7` 的 876 条样本逐条解码后确认：它们全部是带位置 tag 的短视频，内层 `content_type=5`，并不是地图 pin。`45853` 和 `45855` 全量为 `0`，目前没有证据表明这些字段携带经纬度。

另有一项重要修正：部分新版 CDN 数据使用 `45510` 携带第二个 CDN token，而不是旧定义中的 `45504`。如果遇到这类消息，不能仅因旧 Protobuf 定义解码失败就判定消息损坏。
